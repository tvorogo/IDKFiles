⚡ Auto Installer Guide

Auto Installer by °⊥⋊ɹ∀° — version 2.0+ supports flashing via TWRP/PBRP or PC scripts.

Option 1 — Fastboot (PC)
  1. Extract the ROM .zip file
  2. Connect tablet in Fastboot mode
  3. Windows: run install_..._windows.bat (first install) or update_..._windows.bat
  4. Linux: run sudo bash ./install_..._linux.sh or sudo bash ./update_..._linux.sh
Option 2 — Recovery (TWRP/PBRP)
  1. Download the ZIP to your tablet
  2. Boot into TWRP/PBRP recovery
  3. Flash the ZIP and reboot
  4. If first install: reboot to recovery → Factory Reset → Format Data → type yes → reboot
🔵 Bluetooth Fix (null MAC)

Fix 00:00:00:00:00:00 Bluetooth MAC on nabu using Termux.

Step 1 — Install Termux & grant root
  • Download Termux from Play Store
  • KernelSU: open KSU → SuperUser tab → enable for Termux
  • Magisk: run su in Termux, tap Allow in popup
Step 2 — Run commands in Termux 
su
cd
random_mac=$(busybox hexdump -n6 -e '6/1 "%02X"' /dev/urandom | sed 's/\(..\)/\1:/g' | sed 's/:$//')
echo "Generated MAC: $random_mac"
echo $random_mac | sed 's/://g' | xxd -r -p > /sdcard/random_mac.bin
mkdir -p /dev/tmp/persist
mount /dev/block/by-name/persist /dev/tmp/persist
rm -rf /dev/tmp/persist/bluetooth
mkdir -p /dev/tmp/persist/bluetooth
cp /sdcard/random_mac.bin /dev/tmp/persist/bluetooth/.bt_nv.bin
chown 0:0 /dev/tmp/persist/bluetooth/.bt_nv.bin
chmod 0644 /dev/tmp/persist/bluetooth/.bt_nv.bin
chcon u:object_r:vendor_persist_bluetooth_file:s0 /dev/tmp/persist/bluetooth/.bt_nv.bin
chown 0:0 /dev/tmp/persist/bluetooth
chmod 0755 /dev/tmp/persist/bluetooth
chcon u:object_r:mnt_vendor_file:s0 /dev/tmp/persist/bluetooth

If no errors — reboot and verify Bluetooth MAC.

💾 Backup & Restore (Swift Backup)

Use Swift Backup to backup before flashing and restore after.

  1. Grant root to Swift Backup (KSU: SuperUser tab / Magisk: Allow popup)
  2. Create a full backup via Swift Backup
  3. Copy SwiftBackup folder from internal storage to PC or external drive
  4. Flash the ROM
  5. Install Swift Backup from Play Store, grant root
  6. Copy SwiftBackup folder back to internal storage
  7. Open Swift Backup — it auto-detects and restores

⚠ Warning: For Xiaomi Pad 5 (nabu) only. Follow every step precisely.
1

Flash Recovery — ADB Sideload only

Note: Skip if using Auto Installer or Web Flasher.

Boot into fastboot:

adb -d reboot bootloader
fastboot devices
fastboot flash boot boot.img
fastboot flash vendor_boot vendor_boot.img
fastboot flash dtbo dtbo.img
fastboot reboot recovery
⚠ Do not reboot to OS — it will overwrite recovery.

⚠ Chrome / Edge / Opera only — WebUSB is not supported in Firefox or Safari.
  1. Boot your tablet into Fastboot mode (Power + Vol Down)
  2. Connect via USB to your PC
  3. Click Web Flash button on this page
  4. Click Fetch Files — partitions load automatically

❓ Troubleshooting
Device not detected in fastboot / adb
  • Try a different USB cable — many are charge-only
  • Try USB 2.0 port instead of USB 3.0
  • Windows: install Google USB Driver or Universal ADB Driver
  • Enable USB Debugging in Developer Options
Boot loop after flashing
  • Recovery → Factory Reset → Format data
  • Re-flash via ADB sideload
  • Make sure you flashed correct version for nabu
Stuck on first boot
  • First boot can take 5–10 minutes
  • If over 15 min: reboot to recovery, Factory Reset, re-flash
Web Flash — "Array buffer allocation failed"
  • ZIP is too large for browser RAM — switch to Local .img files tab
  • Extract ZIP on your PC first, then select each .img individually
  • This avoids loading the entire ZIP into RAM
Web Flash — other errors
  • Use Chrome, Edge, or Opera — Firefox/Safari don't support WebUSB
  • Device must be in Fastboot mode before clicking Connect
  • Try a different USB cable or port
